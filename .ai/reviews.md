## UX / Design Review

### Critical
- Section order should be: Hero → Services → About → Projects → Contact (trust before portfolio)
- No sticky navigation specified — needed on mobile for long single-page scroll
- No footer defined (phone, email, NIP, copyright)
- No testimonials section — highest-converting element for a local tradesperson
- GDPR gap — contact form collects personal data, Polityka Prywatności is legally required in Poland
- Fixed floating mobile call button ("Zadzwoń") — many visitors won't scroll to contact

### Important
- Image optimisation not specified — unoptimised uploads will kill Lighthouse score (WebP, srcset, lazy load)
- Contact form needs success, error, and loading states
- 404 page missing from roadmap
- Photo lightbox on project detail page not specified
- Project gallery needs service-type filter as portfolio grows

### Nice to have
- Service area callout visible in Hero (not just About)
- WhatsApp link as contact option
- "Back to home" on project detail should be a button, not a text link
- Card grid column strategy needs to be explicit for mobile (1 col → 2 col)

## Technical Architecture Review

### Blockers before implementation
- `better-sqlite3` not in `package.json` — must be added before database-setup
- SQLite schema completely undefined (tables, columns, slug logic, photo ordering column for F-11)
- Photo serving strategy missing — Astro SSR `public/` cannot serve runtime uploads; needs nginx static route or API route
- `better-sqlite3` compiles a native binary — must run `npm install` on the VPS (not copy from Mac)
- Session cookie security unspecified (HttpOnly, Secure, SameSite, expiry, signing)

### Security gaps
- No rate limiting on `/admin` login — trivially bruteforceable without it
- No CSRF protection on admin POST routes
- Photo upload: no file size limit, no MIME type whitelist, no server-side magic byte validation
- `/uploads` directory must be outside project root (not accidentally committed or built)
- `.env` with `ADMIN_PASSWORD` must be in `.gitignore`

### Deployment gaps
- No process manager specified (PM2 or systemd) — app won't survive crashes/reboots
- No reverse proxy specified (nginx/Caddy) — needed for TLS, serving uploads, caching
- HTTPS not mentioned anywhere — session cookies require Secure flag, which requires HTTPS
- No backup strategy for SQLite database + uploads

### Design gaps to resolve before building
- Slug generation: auto from title, manual, or UUID? What on collision? Can it be changed?
- Cover photo: how is it selected (first uploaded, manually designated)?
- Photo deletion: must delete file from disk, not just DB record
- Input validation: title length, description sanitisation, service_type enum enforcement (Zod)

## Business / Marketing Review

### Biggest risk
The site will launch without LocalBusiness structured data or a Google Business Profile strategy, meaning it will rank only for "Allross" (unknown brand) and receive zero organic traffic. All leads will depend on word-of-mouth.

### Critical for Polish market
- `LocalBusiness` / `HVACBusiness` JSON-LD structured data — required for Google Local Pack eligibility
- Google Business Profile setup — drives more calls than the website itself; missing from roadmap
- Privacy policy (RODO) — legally required even for sole traders collecting form data
- F-gazowe certification badge — legally required for refrigerant work, major trust signal
- NIP / firma info — required by B2B clients (building managers, developers) for VAT invoices

### SEO gaps
- Single page = single title tag = can't rank for multiple service+city queries
- Individual city names (Gdańsk, Gdynia, Sopot) must appear in copy, not just "Trójmiasto"
- Per-service pages (future Phase 6) would unlock: "montaż klimatyzacji Gdańsk", "pompa ciepła Gdynia", etc.
- Project titles and descriptions should be written with keywords in mind (admin UI should hint at this)

### Copy and conversion improvements
- "Bezpłatna wycena" as the primary CTA label (not generic "contact")
- Response time promise ("Odpowiadamy w ciągu 24h")
- Testimonials / opinie (even 3 static quotes)
- WhatsApp alongside phone and email
- Service area: explicit city list beyond "Trójmiasto"
